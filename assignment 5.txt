Below is a clear, complete, exam-ready exploration of JavaScript array methods:

---

 üîπ Sample Array (used in all examples)

```js
let numbers = [1, 2, 3, 4, 5];
```

---

 1Ô∏è‚É£ map()

 Purpose

Creates a new array by modifying each element.

 Syntax

```js
array.map((value, index, array) => newValue);
```

 Example: Square each number

```js
let squares = numbers.map(n => n * n);
console.log(squares); // [1, 4, 9, 16, 25]
```

 Key points

‚úî Returns a new array
‚úî Does not change original array
‚úî Used for transformation

---

 2Ô∏è‚É£ forEach()

 Purpose

Executes a function for **each element** (no return).

 Syntax

```js
array.forEach((value, index, array) => {});
```

 Example: Print elements

```js
numbers.forEach(n => console.log(n));
```

 Key points

‚ùå No return value
‚úî Used for **side effects** (logging, DOM update)

---

 3Ô∏è‚É£ filter()

### üëâ Purpose

Creates a **new array** with elements that pass a condition.

 Syntax

```js
array.filter((value, index, array) => condition);
```

 Example: Get even numbers

```js
let evens = numbers.filter(n => n % 2 === 0);
console.log(evens); // [2, 4]
```

 Key points

‚úî Returns a new array
‚úî Used for **selection**

---

 4Ô∏è‚É£ sort()

 Purpose

Sorts array elements.

 Syntax

```js
array.sort((a, b) => a - b);
```

 Example: Sort numbers ascending

```js
let nums = [10, 5, 20, 1];
nums.sort((a, b) => a - b);
console.log(nums); // [1, 5, 10, 20]
```

 Example: Sort strings

```js
let names = ["Banana", "Apple", "Mango"];
names.sort();
console.log(names);
```

 Key points

‚ö†Ô∏è Modifies original array
‚úî Needs compare function for numbers

---

 5Ô∏è‚É£ reduce() (Accumulator)

 Purpose

Reduces array to **one single value**.

 Syntax

```js
array.reduce((accumulator, value) => newAccumulator, initialValue);
```

 Example: Sum of elements

```js
let sum = numbers.reduce((total, n) => total + n, 0);
console.log(sum); // 15
```

 Example: Find maximum

```js
let max = numbers.reduce((a, b) => a > b ? a : b);
console.log(max); // 5
```

 Key points

‚úî Returns a single value
‚úî Used for **sum, max, count, grouping**

---

 6Ô∏è‚É£ COMBINED REAL-WORLD EXAMPLE

### Task:

> Double even numbers and sum them

```js
let result = numbers
  .filter(n => n % 2 === 0)
  .map(n => n * 2)
  .reduce((sum, n) => sum + n, 0);

console.log(result); // 12
```

---

 7Ô∏è‚É£ QUICK COMPARISON TABLE

| Method     |  Returns          | Modifies Original       |  Use Case     |
| -------      | ------------     | -----------------         | ------------ |
| map          |    New array    | ‚ùå No              | Transform    |
| forEach  | Nothing      | ‚ùå No              | Side effects |
| filter   | New array    | ‚ùå No              | Selection    |
| sort     | Same array   | ‚úÖ Yes             | Ordering     |
| reduce   | Single value | ‚ùå No              | Accumulation |

---

 8Ô∏è‚É£ EXAM-READY ONE-LINE DEFINITIONS

* **map()** ‚Üí transforms array elements
* **forEach()** ‚Üí loops without return
* **filter()** ‚Üí selects elements
* **sort()** ‚Üí arranges elements
* **reduce()** ‚Üí reduces to one value

---

 SUMMARY

* Use **map** to change values
* Use **filter** to remove unwanted data
* Use **reduce** to calculate results
* Use **forEach** for actions
* Use **sort** carefully with numbers

# üß© EVENTS IN JAVASCRIPT (Keyboard & Mouse)

## 1Ô∏è‚É£ What is an Event?

An event is an action performed by the user or browser:

* Keyboard press
* Mouse click
* Mouse movement
* Page load

JavaScript listens for these events and responds   to them.

---

 KEYBOARD EVENTS

 Common Keyboard Events

| Event      | Description              |
| ---------- | ------------------------ |
| `keydown`  | Key is pressed           |
| `keyup`    | Key is released          |
| `keypress` | Key pressed (deprecated) |

---

 Keyboard Event Example

```html
<input id="box" placeholder="Type here">

<script>
document.getElementById("box")
  .addEventListener("keydown", function(event) {
    console.log("Key pressed:", event.key);
  });
</script>
```

---

 Keyboard Event Properties

| Property         | Meaning        | Example          |
| ---------------- | -------------- | ---------------- |
| `event.key`      | Key value      | `"a"`, `"Enter"` |
| `event.code`     | Physical key   | `"KeyA"`         |
| `event.ctrlKey`  | Ctrl pressed?  | true / false     |
| `event.shiftKey` | Shift pressed? | true / false     |
| `event.altKey`   | Alt pressed?   | true / false     |

 Example

```js
document.addEventListener("keydown", e => {
  console.log(e.key, e.ctrlKey);
});
```

---

 MOUSE EVENTS

 Common Mouse Events

| Event       | Description     |
| ----------- | --------------- |
| `click`     | Mouse click     |
| `dblclick`  | Double click    |
| `mousedown` | Button pressed  |
| `mouseup`   | Button released |
| `mousemove` | Mouse moves     |
| `mouseover` | Enters element  |
| `mouseout`  | Leaves element  |

---

 Mouse Event Example

```html
<button id="btn">Click Me</button>

<script>
document.getElementById("btn")
  .addEventListener("click", () => {
    alert("Button clicked");
  });
</script>
```

---

 Mouse Event Properties

| Property        | Meaning         |
| --------------- | --------------- |
| `event.clientX` | X position      |
| `event.clientY` | Y position      |
| `event.button`  | Mouse button    |
| `event.target`  | Clicked element |

 Example

```js
document.addEventListener("mousemove", e => {
  console.log(e.clientX, e.clientY);
});
```

---

 addEventListener()

 Syntax

```js
element.addEventListener(event, function, useCapture);
```

 Example

```js
button.addEventListener("click", function() {
  console.log("Clicked");
});
```

 Advantages

‚úî Multiple events allowed
‚úî Cleaner than inline events
‚úî Better control

---

 EVENT OBJECT (KEY PROPERTIES)

 Common Event Properties

| Property                  | Use                 |
| ------------------------- | ------------------- |
| `event.type`              | Event name          |
| `event.target`            | Source element      |
| `event.preventDefault()`  | Stop default action |
| `event.stopPropagation()` | Stop bubbling       |

---

 Prevent Default Example

```html
<a href="https://google.com" id="link">Google</a>

<script>
document.getElementById("link")
  .addEventListener("click", function(e) {
    e.preventDefault();
    alert("Link blocked");
  });
</script>
```

---

 EVENT FLOW (Important)

 1. Capturing phase

 2. Target phase

 3. Bubbling phase

```js
div.addEventListener("click", handler, true); // capturing
div.addEventListener("click", handler, false); // bubbling
```

---

 COMBINED KEY + MOUSE EXAMPLE

```js
document.addEventListener("keydown", e => {
  if (e.key === "Enter") alert("Enter pressed");
});

document.addEventListener("click", e => {
  console.log("Clicked at", e.clientX, e.clientY);
});
```

---

 EXAM-READY SUMMARY

* Keyboard events handle key actions
* Mouse events handle pointer actions
* `addEventListener()` attaches events
* Event object provides details
* `preventDefault()` stops default behavior

---

 ONE-LINE DEFINITIONS

* **Keyboard event** ‚Üí Triggered by key actions
* **Mouse event** ‚Üí Triggered by mouse actions
* **Event object** ‚Üí Contains event details
* **addEventListener** ‚Üí Registers an event handler

 npm and npx ‚Äì Complete Explanation

 1Ô∏è‚É£ What is npm?

npm (Node Package Manager) is a tool used to:

* Install JavaScript libraries (packages)
* Manage dependencies
* Run scripts

It comes automatically with Node.js.

 Example

```bash
node -v
npm -v
```

---

 2Ô∏è‚É£ What is npx?

**npx (Node Package Execute)** is a tool used to:

* Execute Node packages **without installing them globally**
* Run one-time or temporary tools

It also comes with npm (v5.2+).

---

 3Ô∏è‚É£ Why npm is needed

Without npm:

* You must manually download libraries

With npm:

* One command installs everything
* Version control is easy
* Dependency management is automatic
